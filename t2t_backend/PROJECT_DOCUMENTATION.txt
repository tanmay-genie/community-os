  1. PROJECT OVERVIEW
================================================================================

The T2T Backend is a production-grade protocol server for secure,
policy-gated, auditable communication between AI Digital Twins.

Key Principles:
  - Every message passes through a strict multi-gate pipeline
  - RBAC + ABAC policy enforcement at two gates (pre-routing + pre-execution)
  - Immutable audit trail for every action
  - Ed25519 cryptographic message signing for non-repudiation
  - WebSocket real-time push with polling fallback
  - Cross-organization contract and redaction support
  - Loop detection to prevent infinite message cycles

Tech Stack:
  - FastAPI (async Python web framework)
  - SQLAlchemy 2.0 (async ORM with PostgreSQL)
  - Redis (idempotency, caching, loop detection, memory)
  - PyNaCl (Ed25519 signing via libsodium)
  - Pydantic v2 (strict schema validation)
  - Bcrypt (API key hashing)

  5. policy/ — POLICY ENGINE (RBAC + ABAC + CONTRACTS + REDACTION)
================================================================================

Two-gate policy enforcement. Every message is checked TWICE:
  Gate 1: Pre-routing (in send_message)
  Gate 2: Pre-execution (in reply_message when CONFIRM is received)

================================================================================
  8. audit/ — IMMUTABLE AUDIT LEDGER
================================================================================

Every action in the system is recorded. The audit trail is append-only.
