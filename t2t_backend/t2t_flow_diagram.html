Twin-to-Twin
Communication Flow

Human → Twin → SID → Policy → Router → Target Twin → Orchestrator → Audit → Notify

STEP 01
👤 Human User
Entry point. Human gives a command via voice, hotkey, UI action, or background context event.
Voice Input Hotkey UI Action Context Event
Trigger
STEP 02
🤖 Personal Twin
Each human has one twin. It receives the command, wraps it into a structured intent message (T2T Envelope), and forwards to SID for verification.
Intent Creation Envelope Wrap Human Counterpart
Identity Check
STEP 03
🔐 SID Service
Validates twin identity, checks device trust, confirms clearance level. Issues a signed verification token. If invalid → BLOCKED immediately.
Identity Verify Device Trust Clearance Check Revocation Check
Verified Token
STEP 04 — GATE 1
⚖️ Policy Engine (First Gate)
First major checkpoint. Checks: RBAC (role permission), ABAC (clearance + context), org boundary, budget threshold, risk level.
RBAC ABAC Budget Threshold Org Boundary Redaction Profile
Decision
✓ Role + Scope OK
📦 Create Message Envelope
Builds the signed T2T JSON envelope: message_id, thread_id, from/to twin, intent type, payload, scope, security signature (ed25519), idempotency_key.
Signed Idempotent
⚠ Threshold Exceeded
🚨 Escalation Required
Creates an escalation task with full context. Notifies human for approval. SLA timer starts. Approved → resumes flow. Denied → Audit Log: DENIED.
Human Approval SLA Timer
Route Message
STEP 05
🔀 Agent Router / T2T Message Bus
Receives the signed envelope. Validates schema, enforces tenant boundary, rate limits, deduplication, thread ordering. Delivers to target twin via WebSocket or Inbox.
Schema Validation Tenant Boundary Dedup + Retry Thread Order Loop Detection
Deliver
STEP 06
🤖 Target Human Twin
Receives message. Acknowledges (ACK). Re-validates scope for its own context. May auto-confirm, negotiate, or escalate based on its own thresholds.
ACK Scope Re-Check CONFIRM / DECLINE Escalate if needed
Policy Re-Validate
STEP 07 — GATE 2
⚖️ Policy Re-Validation (Second Gate)
Step-level policy check at the target side before execution begins. Ensures the action is still valid in the current context.
Step-Level Auth Context Check
Decision
✓ Allowed
⚙️ Orchestration Layer
Converts confirmed intent into workflow steps. Plans execution. Acquires idempotency lock. Runs the Execution Engine.
Workflow Plan Idempotency Lock
✗ Denied
📋 Audit Log: DENIED
Logs denial event with rule ID, reason, twin ID, timestamp. Flow terminates. Notification sent to originating twin.
POLICY_DENIED Immutable Log
Execute
STEP 08
⚡ Execution Engine
Runs each workflow step using tool adapters. If an error occurs → triggers Rollback/Compensation Logic. On success → updates Memory Layer.
Jira / CRM / Docs Calendar Feature Flags Rollback on Error Memory Update
Log Everything
STEP 09
📜 Audit Log Service
Immutable ledger. Every state transition is recorded: who, what, when, why, policy decision. Fully replayable. Legal holds supported.
Immutable Replayable Non-Repudiation Compliance
Notify
STEP 10
🔔 Notification Service
Sends structured notifications: informational updates, action-required alerts, escalation notices, SLA breach warnings to relevant twins/humans.
Informational Action Required Escalation SLA Breach
